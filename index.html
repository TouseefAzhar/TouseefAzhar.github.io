Introduction to Human–Computer Interaction (HCI)

Human–Computer Interaction (HCI) focuses on the design, evaluation, and implementation of interactive systems for human use. It studies how users interact with systems and how interfaces can be designed to support users’ tasks efficiently.

The main goal of HCI is to improve usability, reduce errors, and enhance user satisfaction by applying design principles and understanding human behavior.

Norman’s Action Cycle

Norman’s Action Cycle explains how users interact with a system through stages such as goal formation, intention, action execution, perception, interpretation, and evaluation.

It introduces two key concepts: Gulf of Execution (difficulty in performing actions) and Gulf of Evaluation (difficulty in understanding system feedback).

GOMS Model

GOMS stands for Goals, Operators, Methods, and Selection Rules. It is used to analyze expert user performance by breaking tasks into structured components.

  • Goals: The objectives the user wants to achieve.
  • Operators: Basic actions like typing or clicking.
  • Methods: Procedures used to accomplish goals.
  • Selection Rules: Rules for choosing between methods.

Keystroke-Level Model (KLM)

KLM is a simplified version of GOMS that predicts task execution time. It focuses on low-level actions such as keystrokes, pointing, mouse clicks, and mental preparation time.

The model assumes expert users and error-free performance, making it useful for estimating efficiency in interface design.
